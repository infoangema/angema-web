1. Descripción funcional por plan

En esta sección se detallan las funcionalidades contempladas en cada versión del sistema de administración de SKUs (Stock Keeping Units). Cada plan suma capacidades sobre el anterior, ofreciendo soluciones adaptadas al tamaño y necesidades del cliente.

1.1 Plan Básico

Resumen: El plan Básico se enfoca en las necesidades fundamentales de gestión de inventario para un solo comercio pequeño.

CRUD de SKUs

Permite Crear, Leer, Actualizar y Eliminar productos (SKUs) en el catálogo. Cada SKU tendrá atributos básicos y un código único generado con lógica basada en sector, tipo, color y otros metadatos.

Manejo de stock

Registro del stock disponible por producto, con posibilidad de ajustar entradas y salidas de inventario. Incluye campo de stock mínimo configurable para controlar umbrales de reposición.

Gestión de pedidos básica

Integración inicial con la tienda online del cliente. Conexión a API de TiendaNube para obtener información de pedidos y a Mercado Pago para verificar pagos.

Login genérico

Autenticación de usuario simple (un solo rol de administrador). Acceso mediante email y contraseña utilizando Firebase Authentication.

Conexión a Firebase

Almacenamiento de datos en Cloud Firestore para aprovechar sincronización en tiempo real y almacenamiento en la nube.

Alertas por stock mínimo

Alerta visible en el panel principal y posibilidad de email automático al administrador cuando el stock baja del mínimo establecido.

1.2 Plan Medio

Resumen: El plan Medio incluye todas las funcionalidades del Básico y agrega características enfocadas en gestión multiusuario y análisis de ventas.

Roles de usuario y gestión de permisos

Sistema de Roles: Administrador, Editor y Usuario (consultor). Interfaz para que el administrador asigne permisos a cada usuario registrado.

Métricas de ventas y dashboard

Dashboard con visualización de datos agregados: ventas totales por período, producto más vendido, ingreso total por categoría, desglose por canal de venta.

Gestión de ventas multicanal

Incorporación de múltiples canales de venta. Consolidación de ventas de tienda física y tienda online para una visión unificada del inventario.

Integración mejorada

Mayor automatización en tiempo real. Configuración de webhooks de TiendaNube para actualización automática de pedidos y stock.

Historial básico de cambios

Registro de modificaciones en inventario. Cada cambio guarda qué usuario hizo el cambio, qué cambió y cuándo, para auditoría básica.

Generación de informes

Exportación de listados de productos, stock y ventas a formatos Excel/PDF para informes gerenciales.

1.3 Plan Avanzado

Resumen: El plan Avanzado incluye todas las características de la versión Media y añade capacidades orientadas a un modelo SaaS multi-tenant.

Arquitectura Multi-Tenant (SaaS)

Soporte para múltiples comercios independientes en la misma plataforma. Datos separados lógicamente utilizando identificador de tenant.

Gestión avanzada de usuarios

Vinculación de usuarios a tenants específicos. Usuario propietario por comercio y Super Administradores de plataforma.

Escalabilidad y aislamiento

Infraestructura planificada para alto volumen con mitigación de efecto "noisy neighbor". Índices y cuotas optimizadas.

Auditoría completa

Historial detallado e inmutable de todas las acciones relevantes. Interfaz de consulta y exportación de reportes de auditoría.

Notificaciones automáticas

Envío automático de correos a proveedores para reabastecimiento basado en stock mínimo. Correos pre-formateados con detalles del producto.

Aplicación móvil (opcional)

Desarrollo de app móvil complementaria con Flutter para Android/iOS. Escaneo de códigos de barras, notificaciones push en tiempo real.

2. Casos de uso por versión

Análisis detallado de los casos de uso típicos para cada plan del sistema de administración de SKUs.

Plan Básico - Casos de uso típicos

Comercio pequeño

Tienda con 1-2 empleados, catálogo de hasta 500 productos, un solo punto de venta físico y tienda online básica.

Emprendimiento digital

Negocio que está digitalizando su inventario por primera vez, necesita control básico de stock y sincronización con e-commerce.

E-commerce simple

Tienda online con TiendaNube que requiere sincronización automática de productos y stock, alertas de reposición.

Plan Medio - Casos de uso típicos

Equipo colaborativo

Comercio con 3-10 empleados que necesitan diferentes niveles de acceso: administradores, editores y consultores.

Análisis de ventas

Negocio que requiere métricas detalladas para tomar decisiones estratégicas basadas en datos de ventas.

Multicanal

Comercio con tienda física y online que necesita consolidación de datos y gestión unificada de inventario.

Plan Avanzado - Casos de uso típicos

Múltiples sucursales

Cadena de tiendas que necesita gestión centralizada con datos separados por ubicación y auditoría completa.

Proveedor de servicios

Empresa que quiere ofrecer el sistema como servicio (SaaS) a múltiples clientes independientes.

Escalabilidad empresarial

Organización con planes de crecimiento acelerado que requiere infraestructura escalable y app móvil.

6. Recomendaciones técnicas y mejoras opcionales

Framework y arquitectura front-end

Angular (versión moderna) con Angular Material o PrimeNG para el design system.

Flutter para móvil

Ideal para app complementaria. Progressive Web App (PWA) considerado para Angular app.

Base de datos y rendimiento

Firestore con diseño optimizado (denormalización, índices). Server-Side Rendering opcional.

Seguridad y autenticación

OAuth social login, 2FA opcionales. Pruebas automatizadas: Unit tests, tests de integración.

Pipeline CI/CD

GitHub Actions para linting, tests, despliegues. Escalabilidad horizontal con Cloud Run.

Inteligencia de inventario

Forecast de demanda (futuro). Soporte multi-moneda e impuestos con campos de moneda.

7. Infraestructura sugerida

Front-end Web

Angular + SSR (opcional Firebase Hosting con Cloud Functions para SSR)

Backend como servicio - Firebase/Google Cloud

  • Base de datos: Cloud Firestore (modo nativo)
  • Autenticación: Firebase Auth
  • Funciones serverless: Cloud Functions for Firebase (Node.js)
  • Hosting y CDN: Firebase Hosting
  • Cloud Storage: Firebase Storage (para imágenes, PDFs)
  • Notificaciones: Firebase Cloud Messaging (FCM)

Opcionales de Google Cloud (plan avanzado)

  • Cloud Run / Cloud Functions 2da gen: Para tareas más intensivas
  • BigQuery (analytics): Para análisis de datos a gran escala
  • Google Identity Platform: Para aislamiento de autenticación multi-tenancy

Integración con terceros

Cloud Functions para llamar APIs de Mercado Pago, TiendaNube. Servicios administrados con alta disponibilidad.

10. Secuencia de acciones principales

Flujos de trabajo típicos en el sistema de administración de SKUs.

Flujo 1: Creación de nuevo SKU

1. Usuario accede al módulo de productos
2. Completa formulario con datos del producto
3. Sistema genera código SKU automáticamente
4. Validación de datos y guardado en Firestore
5. Sincronización con TiendaNube (si configurado)
6. Registro en auditoría (plan medio/avanzado)

Flujo 2: Procesamiento de pedido

1. Webhook recibe pedido de TiendaNube
2. Sistema valida disponibilidad de stock
3. Descuenta stock automáticamente
4. Registra movimiento de stock
5. Verifica stock mínimo y genera alertas
6. Actualiza métricas de ventas (plan medio/avanzado)

Flujo 3: Alerta de stock mínimo

1. Sistema detecta stock por debajo del mínimo
2. Genera notificación en dashboard
3. Envía email al administrador (plan básico)
4. Push notification en app móvil (plan avanzado)
5. Email automático a proveedor (plan avanzado)
6. Registro en sistema de auditoría

11. Wireframes e ideas visuales

Conceptos de diseño y estructura visual para las principales interfaces del sistema.

Panel de control (Dashboard)

Tarjetas con métricas clave (Total ítems, Alertas stock bajo, Ventas por entregar), gráficos de detalle por almacén, lista de productos más vendidos.

Crear nuevo producto (SKU)

Modal con secciones: "Basic Information" (nombre, ubicación, proveedor, código, SKU autogenerado/manual, código de barras), "Media" (subir imágenes), "Sale Information" (precio, profit, margin), "Inventory" (cantidad inicial, unidad). Botón "Advanced settings" para stock mínimo, etc.

Listado de productos

Tabla con columnas (Código, Nombre, Categoría, Stock Actual, Stock Mínimo, Proveedor, Precio), filtros, botón de exportar.

Detalle de producto

Vista con foto, descripción, histórico de movimientos, botón para ajustar stock, gráfico de ventas.

Gestión de usuarios

Lista de usuarios, roles, última actividad. Botón "Invitar usuario".

Módulo pedidos

Lista de pedidos (NroPedido, Fecha, Cliente, Total, Estado), filtros. Detalle de pedido.

Consideraciones de diseño

Responsive design

Adaptación a móviles (menú hamburguesa, tarjetas verticales).

Colores y estilo

Profesional, neutro (azul, gris), indicadores de estado (verde/rojo/amarillo).
