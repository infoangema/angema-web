1. Descripción funcional por plan

En esta sección se detallan las funcionalidades contempladas en cada versión del sistema de administración de SKUs (Stock Keeping Units). Cada plan (Básico, Medio, Avanzado) suma capacidades sobre el anterior, ofreciendo soluciones adaptadas al tamaño y necesidades del cliente.

1.1 Plan Básico

Resumen: El plan Básico se enfoca en las necesidades fundamentales de gestión de inventario para un solo comercio pequeño. Incluye funcionalidades esenciales para crear y manejar SKUs, controlar stock de productos y automatizar tareas básicas de venta.

  • CRUD de SKUs: Permite Crear, Leer, Actualizar y Eliminar productos (SKUs) en el catálogo.
  • Manejo de stock: Registro del stock disponible por producto, con posibilidad de ajustar entradas y salidas de inventario.
  • Gestión de pedidos básica: Integración con API de TiendaNube para obtener información de pedidos y Mercado Pago para verificar pagos.
  • Login genérico: Autenticación de usuario simple mediante email y contraseña, utilizando Firebase Authentication.
  • Conexión a Firebase (Firestore): Almacenamiento de datos en Cloud Firestore.
  • Integración con APIs externas: APIs de Mercado Pago y TiendaNube para sincronizar productos, stock y pedidos.
  • Alertas por stock mínimo: Alerta visible en el panel principal y posibilidad de email automático al administrador.
  • Generación automática de códigos SKU: Asistencia en la generación del código SKU siguiendo reglas definidas.

1.2 Plan Medio

Resumen: El plan Medio incluye todas las funcionalidades del Básico y agrega características enfocadas en gestión multiusuario y análisis de ventas.

  • Roles de usuario y gestión de permisos: Implementación de roles (Administrador, Editor, Usuario/Consultor) con interfaz para asignación de permisos.
  • Métricas de ventas y dashboard de análisis: Dashboard con visualización de datos agregados de ventas.
  • Gestión de ventas multicanal: Posibilidad de consolidar ventas de tienda física y tienda online.
  • Integración mejorada con TiendaNube y Mercado Pago: Automatización en tiempo real mediante webhooks.
  • Historial básico de cambios (auditoría inicial): Registro de modificaciones en SKUs y stock.
  • Mejoras en la generación de informes: Exportación de listados de productos, stock y ventas a Excel/PDF.
  • Usabilidad y experiencia: Mejoras en la interfaz para uso colaborativo, diseño responsive.

1.3 Plan Avanzado

Resumen: El plan Avanzado incluye todas las características de la versión Media y añade capacidades orientadas a un modelo SaaS (Software as a Service) multi-tenant.

  • Arquitectura Multi-Tenant (SaaS): Soporte para múltiples comercios independientes en la misma plataforma.
  • Gestión avanzada de usuarios y roles por comercio: Vinculación de usuarios a tenants específicos.
  • Escalabilidad y aislamiento: Infraestructura planificada para alto volumen.
  • Auditoría completa de cambios (historial): Registro detallado e inmutable de todas las acciones relevantes.
  • Notificaciones automáticas de reposición: Envío automático de correos a proveedores para reabastecimiento.
  • Aplicación móvil (opcional con Flutter): Desarrollo de app móvil complementaria para Android/iOS.
  • Infraestructura escalable y personalizada: Opciones de despliegue dedicado para clientes Enterprise.

2. Casos de uso por versión

A continuación se presentan los casos de uso típicos para cada plan, ayudando a identificar cuál se adapta mejor a las necesidades específicas del negocio.

Plan Básico - Ideal para:

  • Comercios pequeños: Tiendas con un solo punto de venta y catálogo de hasta 500 productos.
  • Emprendimientos: Negocios que están digitalizando su inventario por primera vez.
  • Tiendas online simples: E-commerce con TiendaNube que necesitan sincronización básica.
  • Control básico de stock: Negocios que requieren alertas simples de reposición.

Plan Medio - Ideal para:

  • Comercios en crecimiento: Tiendas con múltiples empleados que necesitan diferentes niveles de acceso.
  • Multicanal: Negocios con tienda física y online que requieren consolidación de datos.
  • Análisis de ventas: Comercios que necesitan métricas para tomar decisiones estratégicas.
  • Gestión colaborativa: Equipos que requieren roles y permisos diferenciados.

Plan Avanzado - Ideal para:

  • Empresas con múltiples sucursales: Cadenas de tiendas que necesitan gestión centralizada.
  • Proveedores de servicios: Empresas que quieren ofrecer el sistema como servicio a sus clientes.
  • Escalabilidad empresarial: Organizaciones con planes de crecimiento acelerado.
  • Auditoría completa: Negocios que requieren trazabilidad total de cambios por compliance.

Sugerencia

Se recomienda abordar el proyecto en fases, iniciando con el Plan Básico o Medio para validar resultados y obtener valor rápidamente, y luego reinvertir hacia planes superiores. La arquitectura propuesta permite que la inversión en etapas iniciales sea aprovechable en fases posteriores.
